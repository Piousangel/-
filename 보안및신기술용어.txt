1. Metaverse(메타버스)
-> 기존의 가상현실이라는 용어보다 진보된 개념 웹과 인터넷 등의 가상세계가 현실세계에 흡수된 형태

2. IP Sec(ip security)
-> ip Sec은 안전하지 않은 네트워크 상의 두 컴퓨터 사이에 암호화된
안전한 통신을 제공하는 프로토콜

- AH(Authentication Header)
데이터가 전송 도전에 변조되었는지를 확인 할 수 있도록 데이터의 무결성에 대해 검사한다.

- ESP(Encapsulating Security Payload): 메세지의 암호화를 제공함

- IKE(Internet Key Exchange)
-> ISKMP, SKEME, Oakley알고리즘의 조합으로 두컴퓨터간의 보안 안결을 설정한다.

3. Linked OPen Data
-> 전세계 오픈된 정보를 하나로 묶는 방식으로 link data, open data의
합성어 이다.

4. 하둡(Haddop)  - 맵리듀스를 기반으로하고있음
-> 오픈 소스를 기반으로 한 분산 컴퓨팅 플랫폼이다.
일반 PC급 컴퓨터들로 가상화된 대형 스토리지를 형성함
다양한 소스를 통해 생성된 빅데이터를 효율적으로 저장하고 처리한다.

5. NAT(network adress translation) = 공유기
-> 사설 ip주소를 공인ip주소로 바꿔주는데 사용하는 통신망 주소 변환기이다.
한정된 하나의 공인ip를 여러개의 내부 사설 ip로 변한하여 공인 Ip를 절약하고
외부 침입에 대한 보안성을 높이기 위한 기술이다.

6. 블록체인
-> 블록을 연결한 모음, 블록체인 기술이 쓰인 가상화폐 비트코인
중앙 집중형 서버에 거래 기록을보관하지 않고 거래에 참여하는
모든 사용자에게 거래 내역을 보내주며 거래 때마다 모든 거래참여자들이 정보를
공유하고 이를 대조해 데이터 위조나 변조를 없애도록함

7.안드로이드 
-> 리눅스 운영체제 자바, 코틀린

8. SSL(Secure Soket Layer) https
인터넷을 통해 전달되는 정보 보안의 안전한 거래를 허용하기 위해
Netscape사에서 개발한 인터넷 통신 규약 프로토콜이다.
- HandShake protocol : 클라이언트와 서버의 상호 인증, 암호 알고리즘, 암호키의 사전합의 진행
- Change Cipher Spec protocol : 협상된 Cipher규격과 암호키를 
- Alert protocol : 다양한 에러 메세지를 전달
- Record Protocol : 트랜스포트 계층을 지나기전에 애플리케이션 데이터를 암호화

9. HTTPS
-> 사용자의 웹 브라우저와 시큐어 웹서버 사이에 암호화된 통신채널을
만들기 위해 SSL, TLS를 사용한다.

10. VPN(virtual private Network)
-> 인터넷과 같은 공중망을 이용하여 사설망과 같은 효과를 얻기위한
기술로 기존의 전용선을 이용한 사설망에 비해 월씬 저렴한 비용으로 보다
뛰어난 연결성이 뛰어나면서 안전한 망을 구성할 ㅅ 있음!

11. 디지털 트윈
-> 최적화 하기 위해 가상모델을 만들어 시뮬레이션 돌려봄
설계, 제조, 서비스에 이르는 모든 과정의 효율성을 향상시킬 수 있다.

12. XML
-> HTML의 단점을 보완한 인터넷 언어로 SGML의 복잡한 단점을 개선한
다목적 마크업 언어
웹상에서 구조화된 문서를 상호교환 가능하도록 설계된 웹 표준문서
사용자가 새로운 태그와 속성을 정의할수 있는 확장성을 가진다.

13. Observer 옵저버패턴
-> 한 객체의 상태가 바뀌면 그 객체에 의존하는 다른 객체들한테 연락이
가고 자동으로 내용이 갱신되는 일대다(one to many) 의존성을 가지는 다지안 패턴이다

14. SOAP(Simple Object Access Protocol)
웹서비스를 실제로 이용하기 위한 객체간의 통신규약으로 인터넷을 통하여
웹서비스가 통신할 수 있게 하는 역할을 담당하는 기술이다

15. 정적분석도구
cppcheck: c++코드에 대한 오버플로우, 메모리 누수 등 분석
pmd: 소스 코드에 대한 사용되지 않는 변수, 최적화 되지 않은 코드 등 결함을 유발할 수 있는 코드 검사
checkstyle: 자바 코드가 소스 코드 표주을 따르고 있는지 검사

16. 샘플링 오라클
-> 테스트 오라클은 테스트의 결과가 참인지 거짓인지를 판단하기 위해서
사전에 정의된 참 값을 입력하여 비교하는 기법 및 활동을 말한다.
테스트 오라클 중 하나가 샘플링 오라클

17. Secure Os
-> 컴퓨터 운영체제의 커널에 보안기능을 추가한 것

18. 메쉬업(Mashup)
-> 웹에서 제공하는 정보 및 서비스를 이용하여 새로운 소프트웨어나 서비스
데이터베이스 등을 만드는 기술

19. Data Mining 데이터마이닝
-> 빅데이터 분석 기술 중 대량의 데이터를 분석하여 데이터 속에 내재되어 있는
변수 사이의 상호관계를 규명하여 일정한 패턴을 찾아내는 기법

20. 리팩토링
-> SW를 보다 쉽게 이해할 수 있고 적은 비용으로 수정할 수 있도록 겉으로
보이는 동작의 변화없이 내부구조를 변경하는 것
come smell을 고치고 다듬는과정
목적: SW의 디자인을 개선시킴, SW를 이해하기 쉽게 만듬

21. AJAX를 써라 : javascript를 사용한 비동기 통신기술

22. 헝가리안 표기법
-> 프로그래밍 언어에서 변수 및 함수의 인자 이름 앞에 데이터 타입을 명시하는 코딩 규칙

23. MapReduce
-> 대용량 데이터를 분산 처리하기 위한 목적으로 개발된 프로그래밍 모델
Google에 의해 고안된 기술로써 대표적인 대용량 데이터 처리를 위한 병렬처리 기법을 제공한다
임의의 순서로 정렬된 데이터를 분산처리하고 이를 다시 합치는 과정을 거친다

24. 랜섬웨어(Ransomware)
인터넷 사용자의 컴퓨터에 침입해 내부 문서 파일등을 암호화해 사용자가
열지 못하게 하는 공격으로 암호 해독용 프로그램의 전달을 조건으로
사용자에게 돈을 요구
