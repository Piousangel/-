데이터 엔지니어는 앱 or 웹에서 발생하는 데이터들을 파이프라인을 통해 데이터베이스에 저장하고 대용량의 데이터를 수집, 관리, 유지보수 하는 일을 함 1. 여러 서버 클러스터로 빅데이터를 처리 서버 클러스터란 각기 다른 서버들을 하나로 묶어서 하나의 시스템같이 동작하게 함으로써, 클라이언트들에게 고가용성의 서비스를 제공하는 것을 말합니다. 클러스터로 묶인 한 시스템에 장애가 발생하면, 정보의 제공 포인트는 클러스터로 묶인 다른 정상적인 서버로 이동한다 서버 클러스터는 사용자로 하여금 서버 기반 정보를 지속적이고, 끊이지 않게 제공 받을 수 있게 해야 합니다. 클러스터 기술은 어플리케이션과 서비스장애, 시스템과 하드웨어 장애, 여러기관의 사이트 장애등의 장애를 대비합니다. 서버 클러스터에서 각 서버가 소유하고 관리하는 로컬 장치나 os및 어플리케이션, 서비스들의 복사본을 만들어서 이를 클러스터가 관리합니다. 서버클러스터는 Single quorum device cluster, Majority node set cluster, Local quorum cluster(싱글 노드 클러스터)로 구성되어있습니다. 2. 클러스터 관리, 분산 컴퓨팅, 리소스 관리, Load balancing관련 문제 해결 3.ETL(Extract Transform Load) 4.DW(Data Warehouse)구축 5.BI(Business Intelligence)구축 6.Hadoop, Spark, Kafka 등을 사용해 업무를 합니다. 데이터 수집 및 가공 하둡, SQL 리눅스 쉘스크립트 BI tool, 대시보드 구축, 파이프라인 개발 데이터 엔지니어의 업무는 주어진 환경에서 최상의 퍼포먼스를 낼 수 있도록 파이프라인을 구축하는 것 특정 기술을 사용할 때 비슷한 도구(툴, 프레임워크) 대신 왜 이도구를 선택했는가 항상 고맨해보는 습관이 필요 데이터 수집 -Apache Kafka, fluntd, Embulk, lgstash 데이터 저장 -HDFS, json, AWS S3, RDB, NoSql등 리눅스 클라우드 서비스 -AWS, GCP, Azure, IBM 대쉬보드(DashBoard) -Metabase, Superset, Zepplin, Redash Docker 데이터 파이프라인 구축 파이프라인 : 데이터 수집 및 가공, 저장을 포함하는 일련의 과정 ETL Extract Transform Load를 수행 TDD(Test-driven-development) 테스트 주도개발 - 에자일과 마찬가지로 매우짧은개발사이클을반복하는소프트웨어개발프로세스중하나이다 개발자는 바라는향상 또는 새로운함수를 정의하는 자동화된테스트테이스를 작성하고 그케이스를 통과하기위한 최소한의양의 코드를생성한다 코드를 표준에맞게 리팩토링즉,테스트가주가되는개발이다
